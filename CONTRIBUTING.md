# Contributing to Chrono

Thanks for your interest in contributing! 🎉

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone <your-fork-url>`
3. Install dependencies: `npm install`
4. Create a `.env` file with your `GEMINI_API_KEY`
5. Build the project: `npm run build`

## Development Workflow

1. Create a new branch: `git checkout -b feature/your-feature`
2. Make your changes
3. Test locally: `npm run dev`
4. Build: `npm run build`
5. Test the built version: `npm start`
6. Commit your changes (use chrono if you want! 😄)
7. Push and create a Pull Request

## Code Style

- Use TypeScript
- Follow existing code patterns
- Keep functions small and focused
- Add comments for complex logic

## Testing

Before submitting a PR:
- Test with `npm run build && npm start`
- Try different scenarios (empty diff, large diff, etc.)
- Test both with and without Jira prefix

## Reporting Issues

- Use the GitHub issue tracker
- Include steps to reproduce
- Mention your Node.js version and OS

## Feature Requests

Open an issue with:
- Clear description of the feature
- Use cases
- Why it would be useful

Thanks for contributing! 🚀
